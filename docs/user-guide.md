# MentorCareer LMS — Инструкция по пользованию

**URL платформы:** https://lms.nadtocheev.ru
**Админ-панель:** https://lms.nadtocheev.ru/admin

---

## 1. Общее описание

MentorCareer LMS — платформа обучения с системой геймификации. Контент организован в иерархию:

```
Роадмап (учебный путь)
  └── Курсы (модули)
       └── Уроки (занятия с контентом)
```

Студенты проходят уроки, зарабатывают баллы, получают достижения и сертификаты. Администраторы создают и управляют контентом через admin-панель.

**Роли:**
- **Администратор** — создаёт контент, управляет пользователями и настройками
- **Студент** — проходит обучение, отслеживает прогресс

> Самостоятельная регистрация отсутствует. Аккаунты создаёт администратор.

## 2. Для студентов

### 2.1. Вход в систему

1. Откройте https://lms.nadtocheev.ru/login
2. Введите email и пароль, полученные от администратора
3. Нажмите «Войти»

**Забыли пароль?**
1. На странице входа нажмите «Забыли пароль?»
2. Введите email → на почту придёт ссылка для сброса
3. Перейдите по ссылке → задайте новый пароль (минимум 6 символов)

### 2.2. Дашборд (главная страница)

После входа вы попадёте на дашборд, где отображается:

- **Приветствие** с вашим именем
- **Статистика** — 5 карточек:
  - Пройденные уроки
  - Общие баллы
  - Серия дней (streak)
  - Полученные достижения
  - Заработанные сертификаты
- **Учебные пути** — доступные роадмапы
- **Последние достижения** — 5 недавно полученных

### 2.3. Роадмапы

**Страница роадмапов** (`/roadmaps`) показывает все опубликованные учебные пути.

**Страница роадмапа** (`/roadmaps/[slug]`):
- Название и описание
- Общий прогресс (процент завершения)
- Список курсов с индикаторами:
  - Завершён (галочка)
  - В процессе (круг)
  - Заблокирован (замок) — нужно пройти предыдущие курсы

### 2.4. Курсы

**Страница курса** (`/courses/[slug]`):
- Название, описание, оценка времени
- Прогресс: «X/Y уроков (Z%)»
- Список уроков с отметками о прохождении
- Кнопка «Назад к роадмапу»

### 2.5. Прохождение урока

**Страница урока** (`/lessons/[slug]`) содержит:

**Контент урока** — может включать:
| Тип блока | Описание |
|-----------|----------|
| Текст | Форматированный текст (заголовки, списки, выделения) |
| Видео | Встроенный плеер или ссылка (YouTube, Yandex.Disk) |
| Изображение | Картинка с подписью |
| Ссылка | Внешний ресурс (Boosty, Telegram, GitHub и др.) |
| Miro | Встроенная доска для совместной работы |
| Файл | Скачиваемый файл с описанием |

**Заметки** (раскрывающаяся секция):
- Личные заметки к уроку (до 5000 символов)
- Видны только вам
- Можно создавать, редактировать и удалять

**Комментарии**:
- Обсуждение урока с другими студентами
- До 2000 символов на комментарий
- Поддержка вложенных ответов

**Кнопка завершения**:
- «Отметить пройденным» → урок считается завершённым
- Начисляются баллы
- Можно отменить (нажать повторно)

**Навигация**:
- Кнопки «Предыдущий урок» / «Следующий урок» внизу страницы

### 2.6. Система баллов

Баллы начисляются автоматически:

| Действие | Баллы (по умолчанию) |
|----------|---------------------|
| Урок пройден | +10 |
| Курс завершён (все уроки) | +50 (бонус) |
| Роадмап завершён (все курсы) | +200 (бонус) |
| Достижение получено | зависит от достижения |

Баллы отображаются в шапке сайта, на дашборде и в профиле. История начислений доступна в профиле.

### 2.7. Достижения

Достижения выдаются автоматически при выполнении условий:

| Достижение | Условие | Бонус |
|------------|---------|-------|
| Первый шаг | 1 урок пройден | +5 |
| Десятка | 10 уроков | +20 |
| Марафонец | 25 уроков | +50 |
| Курс пройден | 1 курс завершён | +30 |
| Полный стек | 5 курсов завершены | +100 |
| Мастер пути | 1 роадмап завершён | +150 |
| Коллекционер баллов | 500 баллов набрано | +50 |

При получении достижения приходит уведомление и email.

### 2.8. Серии (Streaks)

- Каждый день, когда вы завершаете хотя бы 1 урок, серия увеличивается
- Если пропустить день — серия сбрасывается
- В шапке отображается текущая серия (иконка огня)
- Отслеживается рекорд (самая длинная серия)

### 2.9. Лидерборд

Страница `/leaderboard` — рейтинг студентов по общим баллам:
- Топ-50 студентов
- Ваша строка выделена
- Медали для топ-3 (золото, серебро, бронза)

### 2.10. Сертификаты

Сертификаты выдаются автоматически:
- **За курс** — когда все уроки курса пройдены
- **За роадмап** — когда все курсы роадмапа завершены

Каждый сертификат содержит:
- Название курса/роадмапа
- Ваше имя
- Уникальный номер
- Дату выдачи

Все сертификаты доступны на странице `/certificates`.

### 2.11. Профиль

**Просмотр** (`/profile`):
- Аватар, имя, email, описание
- Статистика (уроки, баллы, достижения)
- Список всех достижений с датами
- Последние 10 транзакций баллов

**Редактирование** (`/profile/edit`):
- Загрузка аватара (PNG/JPG, до 5 МБ)
- Поле «О себе» (до 500 символов)

### 2.12. Уведомления

Иконка колокольчика в шапке показывает количество непрочитанных. Уведомления приходят при:
- Завершении курса/роадмапа
- Получении достижения
- Начислении бонусных баллов

### 2.13. Навигация

**Боковое меню:**
- Дашборд
- Роадмапы
- Сертификаты
- Лидерборд
- Профиль
- Переключатель темы (светлая/тёмная)
- Кнопка выхода

На мобильных устройствах меню открывается через иконку-гамбургер.

## 3. Для администратора

### 3.1. Доступ к админ-панели

1. Откройте https://lms.nadtocheev.ru/admin
2. Войдите с учётными данными администратора
3. Откроется Payload CMS — панель управления контентом

### 3.2. Создание контента

Контент создаётся сверху вниз: Роадмап → Курсы → Уроки.

#### Шаг 1: Создание роадмапа

1. В боковом меню: **Контент → Роадмапы → Создать**
2. Заполните:
   - **Название** (обязательно) — slug генерируется автоматически
   - **Описание** — краткое описание учебного пути
   - **Обложка** — изображение для карточки
   - **Порядок** — позиция в списке (чем меньше, тем выше)
   - **Опубликован** — включите для видимости студентам
3. Сохраните

#### Шаг 2: Создание курсов

1. **Контент → Курсы → Создать**
2. Заполните:
   - **Название** (обязательно)
   - **Описание**
   - **Роадмап** (обязательно) — привяжите к роадмапу
   - **Порядок** — позиция внутри роадмапа
   - **Оценка времени** (часы)
   - **Пререквизиты** — курсы, которые нужно пройти перед этим
   - **Опубликован**
3. Сохраните

#### Шаг 3: Создание уроков

1. **Контент → Уроки → Создать**
2. Заполните:
   - **Название** (обязательно)
   - **Описание** (до 300 символов)
   - **Курс** (обязательно) — привяжите к курсу
   - **Порядок** — позиция внутри курса
   - **Оценка времени** (минуты)
   - **Опубликован**
3. Добавьте **блоки контента** (кнопка «Добавить блок»):

| Тип блока | Поля |
|-----------|------|
| **Текст** | Форматированный текст (визуальный редактор) |
| **Видео** | Название, описание, URL, режим (iframe/ссылка), длительность |
| **Изображение** | Загрузка файла, подпись, alt-текст |
| **Ссылка** | URL, название, описание, платформа (Boosty/Telegram/YouTube/GitHub/Другое) |
| **Miro** | URL доски, название, высота отображения |
| **Файл** | Загрузка файла, название, описание |

4. Сохраните

### 3.3. Настройка достижений

1. **Контент → Достижения → Создать**
2. Заполните:
   - **Название** и **Описание**
   - **Иконка** — изображение-бейдж
   - **Бонус баллов** — сколько баллов за получение
   - **Тип критерия:**
     - `lesson_count` — количество пройденных уроков
     - `course_completion` — завершение курса (конкретного или любого N)
     - `roadmap_completion` — завершение роадмапа
     - `total_points` — набрано N баллов
   - **Значение критерия** — пороговое число
   - **ID сущности** (опционально) — для привязки к конкретному курсу/роадмапу
   - **Активно** — включить для работы
3. Сохраните

Достижения проверяются автоматически при каждом завершении урока.

### 3.4. Управление пользователями

1. **Пользователи → Создать**
2. Заполните:
   - **Email** (обязательно, уникальный)
   - **Пароль** (минимум 6 символов)
   - **Имя** и **Фамилия**
   - **Роль:** Admin или Student
   - **Активен** — отключите для блокировки доступа
3. Сохраните

При создании студенту автоматически отправляется приветственное email (если SMTP настроен).

### 3.5. Настройки платформы

**Настройки → Site Settings:**
- **Название платформы** (по умолчанию: MentorCareer LMS)
- **Описание**
- **Логотип**
- **Баллы:**
  - За урок (по умолчанию: 10)
  - Бонус за курс (по умолчанию: 50)
  - Бонус за роадмап (по умолчанию: 200)

### 3.6. Медиа-библиотека

**Контент → Медиа** — управление загруженными файлами:
- Изображения (PNG, JPG)
- Документы (PDF)
- Видео
- Архивы (ZIP)
- Автоматическая генерация миниатюр (400x300, 768x512)

### 3.7. Мониторинг прогресса

Через админ-панель доступны:
- **User Progress** — кто какие уроки прошёл
- **Points Transactions** — все начисления баллов
- **User Achievements** — кто какие достижения получил
- **Certificates** — выданные сертификаты
- **Streaks** — серии активности студентов
- **Notifications** — отправленные уведомления
- **Comments** — обсуждения в уроках
- **Notes** — заметки студентов (видны только владельцу)

## 4. FAQ

**Q: Как создать первого пользователя на пустой базе?**
A: Откройте `/admin` — Payload предложит создать первого администратора через форму регистрации. Далее создавайте студентов через админ-панель.

**Q: Как запустить seed (тестовые данные)?**
A: Выполните `pnpm seed`. Будут созданы: администратор (`admin@mentorcareer.ru` / `admin123`), 2 тестовых студента, 2 роадмапа с курсами и уроками, 7 достижений.

**Q: Как изменить количество баллов за действия?**
A: Админ-панель → Настройки → Site Settings → секция «Баллы».

**Q: Почему курс заблокирован для студента?**
A: У курса установлены пререквизиты — студент должен сначала пройти все уроки в указанных курсах.

**Q: Как студент получает сертификат?**
A: Автоматически, когда все уроки курса (или все курсы роадмапа) отмечены как пройденные.

**Q: Можно ли отменить прохождение урока?**
A: Да, студент может нажать кнопку повторно — статус переключится обратно. Баллы за повторное прохождение не начисляются (защита от дублей).

**Q: Почему email-уведомления не приходят?**
A: Проверьте настройки SMTP в переменных окружения (`SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`). Без SMTP email выводятся в консоль сервера.
